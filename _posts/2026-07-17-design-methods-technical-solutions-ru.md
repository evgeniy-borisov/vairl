Как искать решение, когда «проб и ошибок» уже мало, а дедлайн не ждёт озарения?

Инженерная культура ответила линейкой методов: от сократовской маевтики до АРИЗ Альтшуллера и формального синтеза принципа действия. Этот пост — **карта toolkit’а** для инженеров, студентов техвузов и всех, кто проектирует поиск решений (в том числе рядом с агентными пайплайнами): не начинать с нуля и понимать, *когда* брать мозговой штурм, а когда — противоречия и информационные фонды.

## Три типа методов {#types}

В учебных сводках методы помечают тремя буквами:

| Тип | Суть | Когда брать |
|-----|------|-------------|
| **и** — интуитивные | Ассоциации, аналогии, групповая генерация | Нужно много сырых идей, прототип ещё «мягкий» |
| **э** — эвристические | Готовый фонд приёмов и рекомендаций | Есть прототип и типовые «дыры», нужен направленный поиск |
| **л** — логические | Ключевые операции формализованы | Нужна воспроизводимая схема синтеза / разрешения противоречий |

**Короткая развилка:** нет структуры задачи → интуитивные (штурм, синектика, фокальные объекты). Есть прототип и противоречие → эвристика / АРИЗ. Нужен синтез принципа действия из эффектов → логические методы школы Половинкина–Глазунова.

### Глоссарий

| Термин | Значение |
|--------|----------|
| **ТР** | техническое решение |
| **ТС** | техническая система |
| **АРИЗ** | алгоритм решения изобретательских задач (ядро ТРИЗ) |
| **ТРИЗ** | теория решения изобретательских задач |
| **ИКР** | идеальный конечный результат |
| **ВПР** | вещественно-полевые ресурсы |
| **Веполь** | модель «вещество + поле» (вепольный анализ Альтшуллера) |

## Идея и последовательность действий {#procedures}

| Название | Основная идея и шаги |
|----------|----------------------|
| **Метод каталога** (№ 9) | Случайные словосочетания стимулируют поиск относительно *прототипа*: слово из словаря → присоединить к названию прототипа → использовать как подсказку. |
| **Синектика** (№ 12) | Постоянная междисциплинарная группа; аналогии и разбор хода решения. Развитие мозгового штурма. |
| **Метод контрольных вопросов** (№ 13, 17, 22, 28, 33) | Готовый список вопросов к себе или к «генераторам» на сеансе штурма / синектики. |
| **Мозговой штурм** (№ 21) | Разделить генерацию и оценку: группа генераторов (6–8) → сеанс → эксперты → отбор и конкретизация. |
| **Метод фокальных объектов** (№ 23) | 4–6 случайных объектов → свойства → соединить с фокальным объектом → подсказки. Модификация каталога. |
| **Гирлянды случайностей и ассоциаций** (№ 47) | Синонимы прототипа → случайные объекты → признаки → комбинации → ассоциации → подсказки. |
| **АРИЗ-85В** (№ 20) | Устранить недостаток через **техническое и физическое противоречие**, опираясь на фонды (стандарты, аналоги, приёмы, эффекты). Шаги: модель задачи и конфликтующая пара → оперативная зона/время и ресурсы (ВПР) → идеальный результат (ИКР) и физ. противоречие → решение на ресурсах → стандарты/приёмы/эффекты → формулировка технического решения. |
| **Систематическая эвристика** (№ 38) | Универсальный алгоритм; каждый этап — эвристическая программа (часто слишком общая для конкретной техники). |
| **Обобщённый эвристический алгоритм** (№ 50) | 17 этапов с массивами: требования к ТР, фонды эффектов и приёмов, процедуры оценки. От потребности до обобщения результата. |
| **Десятичные матрицы поиска** (№ 51) | Матрица 10×10: строки — улучшаемые показатели, столбцы — группы приёмов из патентного анализа. |
| **Вепольный анализ** (№ 53) | Язык: вещество, поле, взаимодействие, «плохое» взаимодействие. Формула-модель → типовые преобразования → ТР. По сути остаётся эвристикой. |
| **Морфологический ящик** (№ 11) | Все комбинации дискретных признаков строения прототипа → выбор лучшей → разработка ТР. |
| **Метод Коллера** (№ 42) | 24 элементарные функции: функция устройства → структура элементарных функций → замена эффектами → выбор лучшей структуры. |
| **Синтез физического принципа действия** (№ 56) | Цепочка эффектов *A → B → C* из базы: от требуемой функции «назад» по причинам до условия остановки. |
| **Энергоинформационный метод** (№ 57) | Как № 56, но эффекты описаны единым языком: \(B_{\mathrm{вх}} = K \cdot B_{\mathrm{вых}}\). |
| **Комплексный метод поиска** (№ 58) | Две ветви: синтез системы и доработка прототипа; фонды типовых ТР, идеальности, форм разрешения противоречий. |
| **Параметрический метод** (№ 59) | Модель прототипа → узловые параметры по знакам частных производных → физ. противоречия → приёмы устранения. |
| **Комбинаторный синтез принципов действия** (№ 60) | Как № 56 на *тернарном* языке (объект–свойство–отношение) с проверкой противоречий существования. |

## Мини-пример: морфологический ящик {#example}

Прототип — **настольная лампа**. Главные признаки с вариантами:

| Признак | Варианты |
|---------|----------|
| Источник света | лампа накаливания / LED / OLED-панель |
| Крепление | основание / струбцина / подвеска |
| Питание | сеть / батарея / USB |
| Управление | выключатель / сенсор / приложение |

Перебираем комбинации (например: LED + струбцина + USB + сенсор) и отбираем те, что бьют в цель — портативность, стоимость, ремонтопригодность. Это и есть морфологический ящик Цвикки в миниатюре: не «озарение», а **полный перебор осмысленных осей**.

Если комбинаций слишком много и мешает противоречие («должно быть ярко *и* не греться»), разумный следующий шаг — эвристика / АРИЗ, а не ещё один штурм.

## Четыре тенденции развития {#trends}

Почти столетие «современных» методов (с 1920-х) укладывается в четыре линии:

1. **Формализация** — от интуитивных к эвристическим и логическим. Принципиально новых интуитивных методов почти не появляется.
2. **Структуризация** — от короткой цепочки этапов к разветвлённым схемам с десятками шагов и итерациями.
3. **Специализация** — от универсальных «на любую творческую задачу» к методам под тип задачи и предметную область.
4. **Информатизация** — рост фондов приёмов, эффектов и типовых решений.

| Ориентир | Объём фонда |
|----------|-------------|
| 1-й вариант АРИЗ | ~40 приёмов |
| Обобщённый эвристический метод | ~420 приёмов |
| Метод концептуального проектирования | ~10 000 ед. информации |
| Фонд А. Вегербауэра (в литературе) | >900 эвристических приёмов |

## Что забрать {#takeaway}

1. Методы поиска ТР — не хаос названий, а **линия**: от стимуляции интуиции к фондам приёмов и формальному синтезу.
2. Для старта достаточно трёх точек опоры: **мозговой штурм** (объём идей), **морфологический ящик** (полнота вариантов), **АРИЗ / противоречия** (когда варианты конфликтуют).
3. Чем «логичнее» метод, тем сильнее он зависит от **информационного фонда** — без базы эффектов и приёмов формальная схема пустеет.

Каталог и описания процедур воспроизведены по учебной сводке «Методы поиска технических решений и тенденции их развития» (приложения с таблицами А1/Я1 и А2/Я2).
