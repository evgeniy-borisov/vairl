Gemma 3 — открытая мультимодальная линейка Google (4B / 12B / 27B). Удобный кандидат для **локального G3-агента**: зрение и язык в одном стеке, дообучение через QLoRA, запуск на скромном GPU.

## Зачем Gemma 3 для агента?

По сравнению с API-only VLM у Gemma 3:

- **Открытые веса** — локальный инференс, свои адаптеры, воспроизводимые эксперименты
- **Мультимодальность** — SigLIP-кодировщик изображений + языковая модель в одном пайплайне
- **Эффективное внимание** — Grouped-Query Attention (GQA), длинный контекст (до 128K)
- **Удобство QLoRA** — 4-bit квантизация + низкоранговые адаптеры на consumer GPU

Типичные роли G3-агента:

| Роль | Пример |
|------|--------|
| Восприятие | Изображение → структурированное описание, OCR, VQA |
| Рассуждение | Следующий шаг из визуального и текстового контекста |
| Заземление инструментов | Наблюдение → вызов tool (с ограничителями) |

## Архитектура (эскиз)

```
[ Пользователь: текст / изображение / файл ]
              │
              ▼
   Gemma 3 VLM (4B–27B)
   • SigLIP кодирует изображение → токены
   • Декодер с GQA + RoPE
              │
              ▼
   [ Петля агента ]
   system prompt → схема инструментов → ответ / действие
              │
              ▼
   Опционально: QLoRA-адаптер (под домен)
```

**Ключевое решение:** отделить **модель** (Gemma 3) от **петли агента** (инструменты, память, критик). Модель заземляет наблюдение; петля задаёт ограничения — тот же принцип, что в нейросимволических пайплайнах VAIRL.

## Путь дообучения (QLoRA)

Из пайплайна CV-курса — минимальный рецепт специализированного G3-агента:

1. **База:** `google/gemma-3-4b-pt` (или 12B при достаточном VRAM)
2. **Квантизация:** 4-bit NF4 через BitsAndBytes (`load_in_4bit`, bfloat16 compute)
3. **LoRA:** rank 8–16 на проекции внимания (`q/k/v/o_proj`, `gate/up/down_proj`)
4. **Обучение:** SFTTrainer (TRL), небольшой доменный датасет в chat-формате
5. **Пример домена:** LaTeX-OCR — изображение формулы → LaTeX-код (VLM + структурированный вывод)

Это не полноценное RL-обучение агента — **SFT адаптера** под узкую задачу внутри более широкого агента.

## Локальный vs облачный инференс

| Режим | Когда | Заметки |
|-------|-------|---------|
| Локально 4B + QLoRA | Прототип, приватность, фиксированная нагрузка | ~16 ГБ VRAM с 4-bit |
| Локально 12B/27B | Качественнее восприятие | Нужен мощнее GPU или квантизация |
| Vertex / AI Studio API | Всплески трафика, без железа | Учёт в реестре ModelWatch |

Перед заявлениями о производительности — прогон на **вашем** железе и запись в `models_registry.yaml` (repro: команда, commit, latency, память).

## Идеи интеграции в агентство

- **Аналитическое агентство:** G3 как модуль восприятия перед символьным планированием (PDDL-пайплайн)
- **Try-on / CV-боты:** понимание изображения + консистентность промптов (см. nudges AG-UI в PA)
- **Демо курса:** LaTeX-OCR, VQA по документам, captioning — каждый как tool за одним агентом

## Ограничения (явно)

- Лицензия Gemma Terms — проверить перед коммерческим распространением весов
- Нет нативного видео в Gemma 3 из коробки — для видео Qwen2.5-VL или API
- Качество tool use зависит от промпта и схемы, не только от размера модели
- Галлюцинации визуальных деталей — верификация критиком или вторым проходом
