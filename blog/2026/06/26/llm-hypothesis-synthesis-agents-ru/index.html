Синтез гипотез локальной LLM для повышения качества агентов

26 June 2026, 17:44 MSK

Улучшение агентов — это не одна идея, а пространство гипотез: изменения в планировании, вызове инструментов, памяти, рефлексии и восстановлении после ошибок. В этой публикации — практический пайплайн, который можно запустить в Google Colab без внешних API.

Зачем синтез гипотез?

Когда мы настраиваем агента, обычно перебираем отдельные трюки: chain-of-thought, ReAct, критик, кэш инструментов. Без явной модели «гипотеза» легко:

  • повторять семантически одинаковые идеи;
  • не фиксировать, что именно проверяем;
  • терять связь между изменением конфигурации и метрикой.

Гипотеза в нашем пайплайне — структурированный объект: утверждение, механизм, ось качества, переменные для эксперимента, метрика, предсказание и критерий опровержения.

Архитектура пайплайна

Контекст задачи
    → локальная LLM (Qwen2.5-7B GGUF на T4)
    → JSON-гипотезы по осям качества агента
    → ChromaDB (эмбеддинги + дедупликация)
    → быстрая оценка (эвристика / бенчмарк)
    → итеративный refine с лучшими кандидатами
    → визуализация пространства (PCA / PaCMAP)

Оси качества агента

Ось Что меняем
tool_use Надёжность вызова инструментов и валидность аргументов
planning Декомпозиция целей и многошаговые планы
memory Удержание контекста в длинных сессиях
reflection Самопроверка и критик перед ответом
error_recovery Поведение после сбоев API и инструментов
latency_tradeoff Баланс скорости и качества

Локальная LLM в Colab

Модель работает на GPU виртуальной машины Colab — данные не уходят во внешний API. На T4 (~16 GB VRAM) комфортно запускается квантизованная 7B-модель через llama-cpp-python.

Если GPU недоступен, ноутбук переключается в демо-режим с шаблонными гипотезами — весь пайплайн (ChromaDB, оценка, визуализация) всё равно проходит от начала до конца.

ChromaDB и дедупликация

Каждая гипотеза сохраняется как эмбеддинг (multilingual-e5-small). Перед добавлением проверяется косинусная близость к уже известным — это снижает повтор генерации «переформулированных» идей в следующих раундах.

Итеративный refine

После первого батча топ-гипотезы по score попадают в контекст следующего запроса к LLM. Модель видит, что уже предложено, и ищет новые направления в пространстве — аналог guided exploration в NAS или эволюционном поиске.

Оценка: от эвристики к бенчмарку

В ноутбуке score_hypothesis — заглушка с весами по осям и проверяемостью. В реальном эксперименте её заменяют на:

  • прогон агента с variables из гипотезы на фиксированном наборе эпизодов;
  • метрики: task_success_rate, tool_call_success_rate, recovery_rate, hallucination_rate;
  • статистический порог для принятия/отклонения гипотезы.

Связь с пространством гипотез

Этот пайплайн — практическое продолжение идеи пространства гипотез и PaCMAP: сначала накапливаем кандидатов, затем проецируем их в 2D для интерпретации кластеров и «белых пятен» поиска.

Ограничения

  • Локальная 7B слабее топовых облачных моделей — компенсируйте жёсткой JSON-схемой и критиком.
  • Эвристический score не заменяет реальный бенчмарк.
  • Сессия Colab ограничена по времени; для долгих экспериментов монтируйте Google Drive.

Попробовать

Откройте ноутбук в Colab, включите Runtime → T4 GPU и пройдите ячейки сверху вниз:

hypothesis-synthesis-agents.ipynb
